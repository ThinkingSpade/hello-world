abacus.exe

Abacus

A data analyst you can interrogate

Data · AI agents · 2026

A data analyst you can interrogate — and it doesn't just answer, it investigates. Ask "why did revenue jump?" and it decomposes the change across every dimension into a waterfall that reconciles to the dollar; ask for cohort retention or an anomaly scan and it computes those too, live, on a 125,000-row warehouse running on SQLite-in-WebAssembly in your browser. A pixel crew of four narrates each step, a public 35-question eval proves the browser engine matches its Python twin to the cent, and pasting your own API key puts a live LLM in the planner's seat — guardrailed so it can never write raw SQL.

STACK

    ↳ How it fitsThe semantic layer defines every metric exactly once for both engines; the deterministic parser and the LLM planner both emit the same plan schema, validated key-by-key before the guarded compiler writes any SQL. Your key stays in your tab — calls go straight from the browser to the provider.

      armada.fleet

      Armada

      A control plane for browser fleets

      Distributed Systems · Agentic Automation · 2026

      A control plane for running fleets of real browsers as a managed service. Operators compose multi-tenant campaigns in a React console, a scheduler spreads them across the day, and every run drives a live Chromium session by handing the page's accessibility tree to a frontier model that replies with one schema-checked action.

      STACK

        ↳ How it fitsUnder the hood is the unglamorous platform that keeps it alive: a scheduler with cron, jitter, and ramp-up, warmed persistent identities pulled from a pool, health-scored proxies that disable themselves, classified failures with alerting, and cost metered against a hard ceiling so a runaway run halts itself. Every action is a strict JSON schema, so the model can only click, fill, scroll, or stop, and I can VNC straight into any live worker to watch it think. All of it on Postgres and Redis, running unattended on a single Mac Mini in the corner.

          churn.ipynb

          Churn Radar

          Telecom customer churn prediction

          Machine learning · 2026

          A model that flags the telecom customers most likely to cancel before they actually do — trained on 7,043 real customer records from the public IBM Telco dataset, then exported to score customers live in your browser, with the exact reasons behind every prediction.

          STACK

            ↳ How it fitsI cleaned the data in pandas, built an explicit one-hot feature matrix, and put logistic regression up against gradient boosting — and the readable model wins (AUC 0.845 vs 0.830). Every prediction is a sum of named coefficients, so the live demo can show exactly why each customer scores the way they do — the model's own arithmetic, not a post-hoc explainer.
